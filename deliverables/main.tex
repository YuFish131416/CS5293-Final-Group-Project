% ── Abstract ──────────────────────────────────────────────────────────
\begin{abstract}
Large Language Models (LLMs) are now routinely deployed in applications that
process untrusted inputs and invoke external tools, creating new attack surfaces
that existing security practices do not cover. This paper surveys prompt
injection attacks---both direct and indirect---and tool-use exploitation in
LLM-integrated systems. We organize the threat space into three attack families
(direct prompt injection, indirect prompt injection, and tool-use/agent
exploitation) and four defense categories (input sanitization, instruction
hierarchy, output monitoring, and architectural isolation). By comparing threat
model assumptions, evaluation methodologies, and practical deployability across
recent work, we find that current defenses remain largely reactive and brittle.
We also identify key open problems, including the absence of standardized
benchmarks and a fundamental tension between model capability and security.
\end{abstract}
